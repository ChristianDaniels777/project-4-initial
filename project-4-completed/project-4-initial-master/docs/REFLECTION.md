# Reflection

Using AI for this project made the setup process much faster because it produced a working starting point for both the `Bag<E>` class and the unit tests. The most useful part was being able to give very specific requirements, like using `ArrayList`, preserving the `Container<E>` interface, and covering iterator behavior in the tests.

Even though the AI generated most of the structure, I still had to review everything carefully. I checked whether the methods matched the assignment, whether the tests actually reflected expected behavior, and whether the code was readable enough to explain later. That review step mattered because AI-generated code can be close to correct while still missing small rubric details.

Overall, AI was helpful as a development assistant, not a replacement for understanding the code. It saved time, but I still needed to verify the logic, refine the output, and make sure I could explain both the prompt strategy and the final implementation.
